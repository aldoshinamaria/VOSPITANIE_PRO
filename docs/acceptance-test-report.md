# Acceptance Test Report

Дата проверки: 2026-06-03.

## Проверки среды

- `npm.cmd run lint`: успешно.
- `npm.cmd run build`: успешно.
- `npm.cmd audit --audit-level=moderate`: 0 уязвимостей.
- `.env.local`: отсутствует.
- Реальный Supabase CRUD: не выполнен, потому что нет `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_ANON_KEY`.

## Пользовательские сценарии

### 1. Создать школу

Результат: форма паспорта школы реализована, обязательные поля валидируются, сохранение идет через `updateState()` и Supabase data-access.

Найденные ошибки: форма показывала сообщение об успешном сохранении до ответа Supabase.

Исправление: сохранение переведено на `await updateState()`, при ошибке сообщение успеха не показывается.

### 2. Добавить социального партнера

Результат: несколько партнеров добавляются в паспорт школы и сохраняются вместе с записью школы в таблицу `partners`.

Найденные ошибки: тот же риск ложного успешного сохранения.

Исправление: исправлено через async-сохранение паспорта школы.

### 3. Добавить мероприятие

Результат: карточка мероприятия содержит поля для КПВР, уровни образования, даты, ответственных, статус и отчет. Месяц определяется по дате начала.

Найденные ошибки: мероприятие могло появиться в UI до ошибки Supabase и остаться визуально как сохраненное.

Исправление: `AppProvider` откатывает оптимистичное состояние при ошибке сохранения.

### 4. Добавить конкурс

Результат: быстрый сценарий конкурса создает обычное мероприятие с модулем `Основные школьные дела`, статусом `planned` и автоматическим определением КПВР по классам.

Найденные ошибки: сообщение успеха появлялось до подтверждения записи в Supabase.

Исправление: сохранение конкурса ожидает `await updateState()`.

### 5. Добавить программу внеурочной деятельности

Результат: программа добавляется в реестр и участвует в плане внеурочной деятельности.

Найденные ошибки: ложная индикация успеха при ошибке Supabase.

Исправление: сохранение и удаление программ переведены на async-сценарий.

### 6-8. Сформировать КПВР НОО / ООО / СОО

Результат: КПВР строится из массива мероприятий, фильтруется по уровню образования и группируется по модулям.

Найденные ошибки: реальное попадание новых записей из Supabase нельзя подтвердить без настроенной базы.

Исправление: кодовая логика проверена сборкой; реальная проверка перенесена в блокеры.

### 9. Сформировать план внеурочной деятельности

Результат: план строится из активных программ, есть расчет часов и охвата.

Найденные ошибки: реальная персистентность новых программ не подтверждена без Supabase env.

Исправление: сохранение программ теперь не подтверждается без успешной записи.

### 10. Экспортировать документы

Результат: DOCX-экспорт КПВР и плана внеурочной деятельности реализован в браузере через `docx`.

Найденные ошибки: экспорт использует текущее состояние приложения; если Supabase не настроен, экспортирует демонстрационное состояние и показывает ошибку загрузки данных.

Исправление: глобальная ошибка Supabase остается видимой в интерфейсе.

## Что работает хорошо

- Архитектура data-access скрывает Supabase от компонентов.
- КПВР строится из мероприятий, без ручного дублирования.
- Конкурс добавляется быстрым сценарием.
- DOCX-экспорт работает из браузера на уровне кода и сборки.
- Ошибки Supabase теперь не маскируются зелеными сообщениями.
