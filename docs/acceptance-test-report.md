# Отчёт приёмочной проверки

Дата проверки: 2026-07-23.

## Итог

Локальный MVP и публичный демонстрационный контур готовы к пилотной демонстрации. Рабочий многопользовательский контур подготовлен на уровне схемы, RLS и data-access, но не введён в эксплуатацию: для проекта ещё не выделен отдельный Supabase и не выполнена проверка с двумя реальными учётными записями.

## Проверенные сквозные сценарии

1. Demo Mode загружает отдельную демонстрационную школу и не изменяет Work Mode.
2. Work Mode запускается с пустой школой и проводит пользователя по чек-листу первого запуска.
3. Документ проходит загрузку, анализ, preview, проверку дублей и только затем явный импорт.
4. История импорта показывает пакет и источник; безопасный откат не удаляет вручную изменённые мероприятия.
5. Мероприятие остаётся единым источником для КПВР, планов, контроля исполнения и отчётов.
6. Карточка мероприятия показывает источник, пакет импорта и переходы к связанным данным.
7. Рабочая программа собирается из данных школы, хранит версии, показывает абзацные изменения и восстанавливает выбранную версию.
8. Проверка соответствия создаёт замечания с переходами к нужным разделам рабочей программы.
9. Рабочая программа экспортируется в DOCX и нативный PDF.

## Изоляция и безопасность

- Demo и Work используют разные ключи localStorage.
- Импорт не меняет `AppState.events` без подтверждения пользователя.
- Неуверенные и неполные записи остаются в preview.
- Откат импорта сравнивает сигнатуру записи и сохраняет мероприятия, изменённые после импорта.
- Серверная схема использует `owner_id`, `school_memberships`, RLS и функции проверки доступа.
- Реальную межшкольную изоляцию ещё необходимо подтвердить на отдельном Supabase-проекте.

## Единственный внешний блокер

В подключённом аккаунте Supabase нет проекта VOSPITANIE_PRO. Доступны только несвязанные проекты, поэтому применять схему к ним небезопасно. Для завершения рабочего многопользовательского пилота нужен отдельный проект и решение по организации/стоимости его создания.

Точный порядок ввода в эксплуатацию приведён в `docs/pilot-runbook.md`.
