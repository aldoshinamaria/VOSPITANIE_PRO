# Document Classification Rules

## 1. Цель классификации

Классификация документов нужна, чтобы Воспитание.PRO понимал назначение загруженного файла до извлечения данных и не смешивал разные типы документов в одном потоке обработки.

Классификация помогает:

- определить нормативный уровень документа;
- выбрать правила анализа;
- понять, какие сущности можно извлекать;
- показать пользователю уверенность распознавания;
- не импортировать данные автоматически;
- сохранить документ как источник для будущих проверок, КПВР, рабочей программы и аналитики.

Классификация не создает операционные сущности. Любые найденные записи должны пройти preview, validation, duplicate detection и user confirmation.

## 2. Уровни документов

### federal

Федеральный уровень. Имеет наивысший приоритет среди образовательных и воспитательных документов после федерального законодательства и ФГОС.

Примеры:

- ФГОС;
- ФОП;
- федеральная рабочая программа воспитания;
- федеральный календарный план воспитательной работы;
- приказы Министерства просвещения;
- федеральные методические рекомендации.

### regional

Региональный уровень. Документы субъекта Российской Федерации, министерства или департамента образования региона.

Примеры:

- региональные планы воспитательной работы;
- региональные рекомендации;
- региональные приказы;
- документы институтов развития образования.

### municipal

Муниципальный уровень. Документы муниципального органа управления образованием, муниципальных методических служб или координаторов.

Примеры:

- муниципальный календарь мероприятий;
- муниципальный план воспитательной работы;
- муниципальные рекомендации;
- письма управления образования.

### school

Локальный уровень школы. Документы образовательной организации.

Примеры:

- рабочая программа воспитания школы;
- КПВР школы;
- планы внеурочной деятельности;
- локальные положения;
- приказы директора;
- отчеты;
- планы объединений.

### unknown

Неопределенный уровень. Используется, если документ не содержит надежных признаков уровня или содержит смешанные признаки.

Такие документы остаются в preview-only режиме и требуют ручной проверки.

## 4. Правила по типам документов

### ФГОС

**Признаки определения**

- упоминание федерального государственного образовательного стандарта;
- номер приказа Минпросвещения или Минобрнауки;
- структура требований к результатам, условиям и программе;
- уровни образования: НОО, ООО, СОО.

**Ключевые слова**

- `ФГОС`;
- `федеральный государственный образовательный стандарт`;
- `требования к результатам`;
- `требования к структуре`;
- `требования к условиям`;
- `начального общего образования`;
- `основного общего образования`;
- `среднего общего образования`.

**Уровень по умолчанию**

`federal`

**Что может извлекаться**

- нормативные требования;
- уровни образования;
- личностные результаты;
- признаки обязательных разделов;
- основания для проверки рабочей программы;
- связи с `FederalRequirement`, `FederalTargetResult`, `ComplianceCheck`.

### ФОП

**Признаки определения**

- название федеральной образовательной программы;
- структура образовательной программы;
- разделы про рабочую программу воспитания;
- связь с календарным планом воспитательной работы.

**Ключевые слова**

- `ФОП`;
- `федеральная образовательная программа`;
- `федеральная основная общеобразовательная программа`;
- `рабочая программа воспитания`;
- `календарный план воспитательной работы`;
- `целевой раздел`;
- `содержательный раздел`;
- `организационный раздел`.

**Уровень по умолчанию**

`federal`

**Что может извлекаться**

- структура программы;
- обязательные разделы;
- требования к воспитательной части;
- целевые ориентиры;
- федеральные направления воспитания;
- связи с `FederalProgramSection`, `FederalDirection`, `WorkProgram`.

### Федеральная рабочая программа воспитания

**Признаки определения**

- прямое название федеральной рабочей программы воспитания;
- описание цели воспитания;
- целевые ориентиры по НОО, ООО, СОО;
- федеральная структура разделов воспитательной программы.

**Ключевые слова**

- `федеральная рабочая программа воспитания`;
- `цель воспитания`;
- `целевые ориентиры`;
- `уклад общеобразовательной организации`;
- `виды, формы и содержание воспитательной деятельности`;
- `кадровое обеспечение`;
- `анализ воспитательного процесса`.

**Уровень по умолчанию**

`federal`

**Что может извлекаться**

- целевые ориентиры;
- направления воспитания;
- обязательные разделы;
- требования к укладу школы;
- структура рабочей программы;
- шаблоны проверок для `WorkProgramComplianceChecker`.

### Федеральный календарный план воспитательной работы

**Признаки определения**

- перечень памятных дат;
- группировка по месяцам;
- федеральные государственные и общественно значимые даты;
- рекомендации для календарного планирования.

**Ключевые слова**

- `федеральный календарный план`;
- `календарный план воспитательной работы`;
- `памятные даты`;
- `День знаний`;
- `День Победы`;
- `День народного единства`;
- `месячник`;
- `акция`.

**Уровень по умолчанию**

`federal`

**Что может извлекаться**

- preview мероприятий;
- даты и месяцы;
- федеральные события;
- признаки традиций;
- направления воспитания;
- возможные записи для КПВР после подтверждения пользователя.

### Стратегия

**Признаки определения**

- долгосрочный документ развития воспитания;
- стратегические цели и задачи;
- описание государственной политики;
- ценностные ориентиры.

**Ключевые слова**

- `стратегия развития воспитания`;
- `стратегические цели`;
- `государственная политика`;
- `ценностные основы`;
- `приоритеты воспитания`;
- `развитие воспитания`.

**Уровень по умолчанию**

`federal`

Если документ выпущен регионом или муниципалитетом, уровень определяется по реквизитам источника.

**Что может извлекаться**

- стратегические направления;
- ценностные основания;
- рекомендации;
- долгосрочные приоритеты;
- связи с `FederalDirection`, `ActivityDirection`, `WorkProgram`.

### Приказ

**Признаки определения**

- реквизиты приказа;
- номер и дата;
- формулировки `приказываю`, `утвердить`, `назначить`;
- приложения к приказу.

**Ключевые слова**

- `приказ`;
- `приказываю`;
- `утвердить`;
- `назначить`;
- `ответственный`;
- `приложение`;
- `№`.

**Уровень по умолчанию**

`unknown`

Уровень определяется по органу-издателю:

- Минпросвещения или федеральный орган — `federal`;
- региональное министерство — `regional`;
- муниципальное управление образования — `municipal`;
- директор школы — `school`.

**Что может извлекаться**

- реквизиты документа;
- утвержденные планы и положения;
- ответственные лица;
- сроки;
- приложения;
- связи с `NormativeDocument`, `DocumentProcessingRecord`, `SchoolEvent` preview.

### Положение

**Признаки определения**

- локальный или нормативный документ, описывающий порядок работы;
- разделы `общие положения`, `цели`, `задачи`, `порядок организации`;
- утверждение приказом.

**Ключевые слова**

- `положение`;
- `общие положения`;
- `цели и задачи`;
- `порядок организации`;
- `участники`;
- `ответственность`;
- `утверждено приказом`.

**Уровень по умолчанию**

`school`

Если положение утверждено региональным или муниципальным органом, уровень определяется по источнику.

**Что может извлекаться**

- правила работы объединений;
- положения о советах, музеях, ЮИД, медиацентре;
- требования к участникам;
- ответственные лица;
- связи с `EducationalSystem`, `EducationalAssociation`, `NormativeDocument`.

**Признаки определения**

- табличная или списочная структура мероприятий;
- сроки;
- ответственные;
- направления или разделы;
- учебный год или период.

**Ключевые слова**

- `план`;
- `мероприятие`;
- `сроки`;
- `ответственные`;
- `классы`;
- `направление`;
- `учебный год`.

**Уровень по умолчанию**

`school`

Если план является федеральным, региональным или муниципальным календарем, уровень определяется по названию и источнику.

**Что может извлекаться**

- preview мероприятий;
- сроки и месяцы;
- ответственные;
- классы;
- направления деятельности;
- связи с `SchoolEvent`, `ActivityDirection`, `ActivityPlan`, `KpvrItem`.

- описание выполненной работы;
- итоги периода;
- показатели, охват, результаты;
- формулировки `проведено`, `приняли участие`, `охват`, `итоги`.

**Ключевые слова**

- `отчет`;
- `итоги`;
- `проведено`;
- `приняли участие`;
- `охват`;
- `результаты`;
- `выполнение плана`.

**Уровень по умолчанию**

`school`

**Что может извлекаться**

- факты проведения мероприятий;

- календарный план как приложение;
- учебный год и название школы.

**Ключевые слова**

- `уклад школы`;
- `целевой раздел`;
- `содержательный раздел`;

- `виды, формы и содержание воспитательной деятельности`;

**Уровень по умолчанию**

### Другое

**Признаки определения**

- документ не соответствует надежно ни одному типу;
- недостаточно текста;
- смешанная структура;
- отсутствуют реквизиты;
- файл содержит только скан без OCR.

**Ключевые слова**

Ключевые слова не определены. Используются общие признаки текста, структуры, таблиц, списков и реквизитов.

- возможные организации;
- предупреждения для ручной проверки.

Операционные сущности из таких документов не создаются без ручной классификации и подтверждения.

## 5. Правила обработки неопределенных документов

Если документ классифицирован как `unknown`, система должна:

1. Показать документ пользователю в preview.
2. Отобразить низкую уверенность классификации.
3. Показать найденные признаки, если они есть.
4. Не создавать события, объединения, партнеров, инфраструктуру или требования автоматически.
5. Не использовать документ как источник обязательных требований.
6. Предложить ручную проверку типа и уровня документа.
7. Хранить документ в `processedDocuments` как результат обработки.
8. Разрешить будущую повторную классификацию.

Если документ поврежден, пустой или требует OCR:

- установить статус проверки качества;
- не запускать импорт;
- не использовать документ в генерации КПВР или рабочей программы;
- показать предупреждения пользователю.

## Общие правила уверенности

### Высокая уверенность

Документ содержит прямое название типа, реквизиты, устойчивые ключевые слова и структурные признаки.

### Средняя уверенность

Документ содержит часть признаков, но не хватает реквизитов или структура смешанная.

### Низкая уверенность

Документ содержит отдельные ключевые слова без надежного контекста.

При низкой уверенности запись остается только в preview и не создает операционные сущности.

## Общие правила безопасности импорта

Классификация документа не равна импорту данных.

Любые извлеченные записи должны пройти:

- preview;
- validation;
- duplicate detection;
- user confirmation.

Если данные противоречат более приоритетному нормативному источнику, нижестоящий документ не должен переопределять вышестоящие требования.
